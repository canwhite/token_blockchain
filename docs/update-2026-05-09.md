# 更新记录 (2026-05-09)

## 本次补充完成的功能

### 1. 幂等性充值 `RechargeWithIdempotency`
**对应原项目**: `AddTokensByEmailWithIdempotency` in `service/user_service.go`

| 原实现 | 本项目实现 |
|--------|-----------|
| Fabric 链码调用 | 轻量区块链 Write |
| MongoDB RechargeRecord 集合 | MongoDB `recharge_records` 集合 |
| 3种状态: pending/success/failed | 3种状态: pending/completed/failed |

```go
// service/user_service.go
func (s *UserService) RechargeWithIdempotency(userId string, amount int, description string, orderSN string) (*database.UserCredit, error)
```

**流程**:
1. 检查订单是否已处理（幂等性检查）
2. 若已 completed → 直接返回已有结果
3. 若 failed → 返回错误
4. 若 pending → 创建充值记录
5. 执行充值
6. 更新充值记录状态

---

### 2. SSE 事件推送 `/api/v1/events/listen`
**对应原项目**: `EventService` + `streamEvents` in `api/server.go`

| 原实现 | 本项目实现 |
|--------|-----------|
| Fabric `network.ChaincodeEvents()` | 轮询 `blockchain.GetBlockchainChain()` |
| 解析 `event.EventName` | 推送 `Block.DataType` 作为事件名 |
| 事件触发 MongoDB 同步 | 服务端推送新区块数据 |

```go
// api/server.go
func (s *Server) streamEvents(c *gin.Context)
```

**实现方式**:
- 通过 SSE (Server-Sent Events) 长连接
- 轮询区块链长度变化，检测新区块
- 推送非 genesis 区块的数据

---

### 3. `GetBlockchainChain` 导出函数
**新增位置**: `blockchain/blockchain.go`

```go
func GetBlockchainChain() []Block
```

**用途**: SSE 事件推送需要读取完整区块链列表，返回副本防止并发问题。
